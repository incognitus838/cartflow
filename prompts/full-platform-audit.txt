## Phase 2 — UI/UX (iPhone / Apple HIG)

Audit and fix against these patterns (see also `prompts/admin-dashboard-ux-audit.txt`):

### Visual & layout
- Clear hierarchy: one primary action per screen; secondary actions visually subdued
- Generous whitespace; 8pt spacing rhythm; rounded corners consistent (`--cf-radius-*`)
- Typography: SF-like scale — large titles, subdued subtitles (`#86868b` muted), semibold labels
- Surfaces: light `#fbfbfd` / `#f5f5f7` backgrounds; subtle borders `black/[0.06]`; no harsh shadows
- Sticky headers with backdrop blur on storefront and dashboard toolbars
- Mobile-first: touch targets ≥ 44px; bottom-safe padding on storefront; horizontal scroll only for charts/pills

### Navigation
- Predictable back paths; no trap screens
- Active nav state (`aria-current="page"`)
- External storefront opens in new tab with clear label

### Feedback
- Every mutation → toast (success/error) via Sonner
- Loading states on buttons (`Saving…`, disabled + opacity)
- Empty states with one clear next action
- Inline validation before network (onboarding, checkout, settings)

### Storefront designer
- Live preview must match customer view (theme, accent, messaging, layout)
- Preview loads fast (no full-catalog fetch — sample products only)
- Save → live store reflects within revalidate window

### Accessibility (non-negotiable)
- Semantic HTML per admin UX audit checklist
- `aria-label` on icon-only buttons
- `aria-pressed` on toggles/pills
- Focus visible on all interactive elements
- Form labels tied to inputs (`htmlFor` / `useId()`)
